**Assignia** (от англ. *assign* - назначать)- система распределения задач по ролям и командам. Обеспечивает видимость учёта рисков и аналитику, отслеживать прогресс и оценивать эффективность продвижения.

## Возможности
- **Ролевая модель:** Владелец, Администратор, Менеджер, Кадровый аналитик, Техподдержка, Участник.
- **Проекты и задачи:** прямое взаимодействие ролей с реальными задачами.
- **Направления и команды:** структурирование прозрачной работы внутри проекта.
- **Риски:** оценка шанса и последствий для каждой задачи.
- **Аналитика:** статистика по командам и участникам, оценка успеваемости.
- **Запросы:** встроенная возможность создать пользовательский запрос, взаимодействовать с техподдержкой, которая поможет решить проблему.
- **Уведомления:** в реальном времени получать актуальную информацию о состоянии.
- **Интерактивный интерфейс:** доступно на множестве устройств.
- **Доступность:** Имеет настроенные исходники для быстрого запуска и дальнейшей конфигурации.

## Запуск 

```bash
#1. Начало пользования
# Переход в рабочее пространство
git clone https://github.com/WhereIU/assignia.git
cd assignia
## или другой репозиторий

#2.1 Запуск разработки
# Конфигурирование для локальной разработки 
cp .env.example .env 
## Конфигурация .env

# Установка разработки
make install

# Запуск для разработки
make run

#2.2 Запуск продакшен
# Конфигурирование для продакшена
cp .env.example .env.prod
## Конфигурация .env.prod

# Установка продакшена 
make install-prod

# Запуск продакшена локально
make run-prod

#3.0 Подготовка к деплою
## Замена nginx сертификатов на свои, включая включение их в nginx/.nginx.conf
## Настройка подключения к серверам по ssh (ansible)

#3.1 Локальный деплой
## Настройка серверов ansible/my_inventory.ini
make deploy-ansible

#3.2 Автоматический деплой
# Новые версии в github ветку main активируют CI/CD
## Для полноценной работы необходимо добавить в github секретах два окружения ci и cd:
## В ci добавляются секрет ENV_FILE_CI (заполненный по .env.example примеру в вид разработки .env),
## В cd окружении добавляются секреты: ENV_FILE_PROD (заполненный по .env.example примеру в вид продакшена .env.prod),
## SSH_PRIVATE_KEY, ANSIBLE_HOSTS (для подключения к удалённым серверам) и сертификаты (по nginx, см. workflow/ci-cd.yml). 
```
